## Overview

The notification system alerts users when workflows require their attention. Notifications are created when human tasks are assigned, and can be customized per workflow step.

## Notification Schema

### Database Schema

```sql
CREATE TABLE notification (
    id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id text NOT NULL REFERENCES "user"(id),
    workflow_instance_id uuid REFERENCES workflow_instance(id),
    step_instance_id uuid REFERENCES step_instance(id),
    notification_type text NOT NULL,
    title text NOT NULL,
    body text NOT NULL,
    is_read boolean NOT NULL DEFAULT false,
    read_at timestamptz,
    created_at timestamptz NOT NULL DEFAULT now()
);
```

### Pydantic Response Model

```python
class NotificationResponse(BaseModel):
    id: UUID
    workflowInstanceId: UUID | None = None
    stepInstanceId: UUID | None = None
    notificationType: str
    title: str
    body: str
    isRead: bool
    readAt: str | None = None
    createdAt: str
```

## Notification Types

Current implementation supports:

- **task_assigned**: User assigned to a new human task
- **task_escalated**: Task escalated to next priority user (future)
- **reminder**: Reminder for pending task (future)
- **workflow_completed**: Workflow the user initiated completed (future)
- **workflow_rejected**: Workflow the user initiated was rejected (future)

<Note>
The platform is designed to support multiple notification types, but currently only `task_assigned` is actively used.
</Note>

## Notification Templates

Workflow authors can customize notification messages per step using templates.

### Template Schema

```sql
CREATE TABLE notification_template (
    id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
    workflow_version_id uuid NOT NULL REFERENCES workflow_definition_version(id),
    step_definition_id uuid REFERENCES workflow_step_definition(id),
    event_type text NOT NULL,
    channel text NOT NULL DEFAULT 'in_app',
    title_template text NOT NULL,
    body_template text NOT NULL,
    allow_actor_override boolean NOT NULL DEFAULT true,
    supported_variables jsonb NOT NULL DEFAULT '[]'::jsonb,
    created_at timestamptz NOT NULL DEFAULT now()
);
```

### Template Model

```python
class WorkflowNotificationTemplateInput(BaseModel):
    titleTemplate: str = ""
    bodyTemplate: str = ""
    allowActorOverride: bool = True
```

### Template Variables

Supported variables for message rendering:

- `{workflowName}`: Workflow definition name
- `{stepLabel}`: Current step label
- `{stepCode}`: Current step code
- `{actorEmail}`: Assigned user's email

### Template Example

```json
{
  "titleTemplate": "Action Required: {stepLabel}",
  "bodyTemplate": "The workflow '{workflowName}' has reached {stepLabel} and requires your review. Assigned to: {actorEmail}",
  "allowActorOverride": true
}
```

### Template Rendering

Implemented in `runtime.py:131-145`:

```python
def _render_notification_template(
    title_template: str | None,
    body_template: str | None,
    context: dict[str, Any],
    fallback_title: str,
    fallback_body: str,
) -> tuple[str, str]:
    title = title_template or fallback_title
    body = body_template or fallback_body

    for key, value in context.items():
        title = title.replace(f"{{{key}}}", str(value))
        body = body.replace(f"{{{key}}}", str(value))

    return title, body
```

<Warning>
Template rendering uses simple string replacement. Variable names are case-sensitive and must match exactly.
</Warning>

## Notification Creation

Notifications are created when tasks are assigned to users.

### Creation Logic

Implemented in `runtime.py:82-128`:

```python
def _create_notification(
    cursor,
    user_id: str,
    workflow_instance_id,
    step_instance_id,
    notification_type: str,
    title: str,
    body: str,
) -> None:
    cursor.execute(
        """INSERT INTO notification (
            user_id,
            workflow_instance_id,
            step_instance_id,
            notification_type,
            title,
            body
        ) VALUES (%s, %s, %s, %s, %s, %s)
        RETURNING id""",
        (user_id, workflow_instance_id, step_instance_id, notification_type, title, body),
    )
    notification_id = cursor.fetchone()["id"]
    
    # Record outbox event for async delivery
    _record_outbox(
        cursor,
        "notification",
        notification_id,
        "notification.created",
        {
            "userId": user_id,
            "workflowInstanceId": str(workflow_instance_id),
            "stepInstanceId": str(step_instance_id),
            "notificationType": notification_type,
            "title": title,
            "body": body,
        },
    )
    
    # Publish realtime event
    publish_realtime_event(
        cursor,
        "runtime.notifications.changed",
        channels=["notifications"],
        users=[user_id],
        workflow_instance_id=str(workflow_instance_id) if workflow_instance_id else None,
        notification_id=str(notification_id),
    )
```

## Notification Delivery

Notifications are delivered through the outbox pattern to ensure reliability.

### Outbox Event

When a notification is created, an outbox event is recorded:

```python
# From runtime.py:43-56
def _record_outbox(
    cursor,
    aggregate_type: str,
    aggregate_id,
    event_type: str,
    payload: dict[str, Any],
) -> None:
    cursor.execute(
        """INSERT INTO outbox_event (aggregate_type, aggregate_id, event_type, payload)
           VALUES (%s, %s, %s, %s)""",
        (aggregate_type, aggregate_id, event_type, Json(payload)),
    )
```

### Outbox Schema

```sql
CREATE TABLE outbox_event (
    id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
    aggregate_type text NOT NULL,
    aggregate_id uuid NOT NULL,
    event_type text NOT NULL,
    payload jsonb NOT NULL DEFAULT '{}'::jsonb,
    headers jsonb NOT NULL DEFAULT '{}'::jsonb,
    status text NOT NULL DEFAULT 'pending',
    available_at timestamptz NOT NULL DEFAULT now(),
    claimed_at timestamptz,
    claimed_by text,
    published_at timestamptz,
    retry_count integer NOT NULL DEFAULT 0,
    max_attempts integer NOT NULL DEFAULT 20,
    last_error text,
    last_error_at timestamptz,
    created_at timestamptz NOT NULL DEFAULT now(),
    updated_at timestamptz NOT NULL DEFAULT now()
);
```

### Delivery Flow

<Steps>
  <Step title="Notification Created">
    Notification row inserted into `notification` table
  </Step>
  
  <Step title="Outbox Event Recorded">
    Event written to `outbox_event` with status `pending`
  </Step>
  
  <Step title="Worker Polls Outbox">
    Background worker queries pending events
  </Step>
  
  <Step title="Event Published">
    Worker publishes event to RabbitMQ
  </Step>
  
  <Step title="Consumer Processes">
    Notification consumer sends in-app notification, email, etc.
  </Step>
  
  <Step title="Event Marked Published">
    Outbox event status updated to `published`
  </Step>
</Steps>

<Note>
The outbox pattern ensures notifications are never lost, even if RabbitMQ is temporarily unavailable.
</Note>

## Notification API

### Get My Notifications

Retrieve notifications for the current user:

```http
GET /api/v1/me/notifications
```

Implementation would query:

```sql
SELECT * FROM notification
WHERE user_id = $1
ORDER BY created_at DESC
LIMIT 50;
```

### Mark as Read

Mark a notification as read:

```http
POST /api/v1/notifications/{notification_id}/read
```

Implementation:

```sql
UPDATE notification
SET is_read = true, read_at = now()
WHERE id = $1 AND user_id = $2;
```

### Unread Count

Get count of unread notifications:

```sql
SELECT COUNT(*) AS unread_count
FROM notification
WHERE user_id = $1 AND is_read = false;
```
