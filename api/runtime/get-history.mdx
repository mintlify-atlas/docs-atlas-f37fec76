---
title: List Workflow Instances
api: GET /api/v1/workflow-instances
---

Retrieves a list of all workflow instances with summary information. Instances are returned in descending order by start time (most recent first).

## Authentication

Requires Bearer token authentication.

## Response

<ResponseField name="items" type="array">
  Array of workflow instance summaries.
  
  <ResponseField name="items[].id" type="UUID">
    Unique identifier for the workflow instance.
  </ResponseField>
  
  <ResponseField name="items[].workflowDefinitionId" type="UUID">
    The workflow definition this instance belongs to.
  </ResponseField>
  
  <ResponseField name="items[].workflowDefinitionKey" type="string">
    The key of the workflow definition.
  </ResponseField>
  
  <ResponseField name="items[].workflowDefinitionName" type="string">
    Human-readable name of the workflow.
  </ResponseField>
  
  <ResponseField name="items[].workflowVersionId" type="UUID">
    The specific version of the workflow definition.
  </ResponseField>
  
  <ResponseField name="items[].businessKey" type="string | null">
    Optional business identifier for tracking.
  </ResponseField>
  
  <ResponseField name="items[].runNumber" type="integer">
    Sequential run number for this business key.
  </ResponseField>
  
  <ResponseField name="items[].status" type="string">
    Current workflow status. One of:
    - `running` - Workflow is actively executing
    - `waiting` - Workflow is waiting for human action
    - `paused` - Workflow is paused (e.g., waiting for subworkflow)
    - `completed` - Workflow completed successfully
    - `rejected` - Workflow was rejected
    - `failed` - Workflow failed due to error
    - `cancelled` - Workflow was cancelled
  </ResponseField>
  
  <ResponseField name="items[].currentStepCode" type="string | null">
    Code identifier of the current step (null if completed).
  </ResponseField>
  
  <ResponseField name="items[].currentStepLabel" type="string | null">
    Human-readable label of the current step.
  </ResponseField>
  
  <ResponseField name="items[].startedBy" type="string">
    User ID of the workflow initiator.
  </ResponseField>
  
  <ResponseField name="items[].startedAt" type="string">
    ISO 8601 timestamp when the workflow started.
  </ResponseField>
  
  <ResponseField name="items[].completedAt" type="string | null">
    ISO 8601 timestamp when the workflow completed (null if still running).
  </ResponseField>
</ResponseField>

## Example Request

<CodeGroup>

```bash cURL
curl -X GET https://api.example.com/api/v1/workflow-instances \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```javascript JavaScript
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN'
  }
});

const data = await response.json();
```

```python Python
import requests

response = requests.get(
    'https://api.example.com/api/v1/workflow-instances',

```

<ResponseField name="200" type="OK">
  Workflow instances retrieved successfully.
</ResponseField>

<ResponseField name="401" type="Unauthorized">
  Missing or invalid authentication token.
</ResponseField>

## Use Cases

### Filtering by Status

While the API returns all instances, you can filter client-side by status:

```javascript
const waitingInstances = data.items.filter(item => item.status === 'waiting');
const completedInstances = data.items.filter(item => item.status === 'completed');
```

### Monitoring Active Workflows

Track workflows that require attention:

```javascript
const activeWorkflows = data.items.filter(item => 
  ['running', 'waiting', 'paused'].includes(item.status)
);
```

### Business Key Tracking

Find all runs for a specific business key:

```javascript
const expenseRuns = data.items.filter(item => 
  item.businessKey === 'EXP-2024-001'
);
```
