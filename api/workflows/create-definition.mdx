---
title: Create Workflow Definition
api: POST /api/v1/workflow-definitions
---

Creates a new workflow definition with an initial draft version. This establishes the structure, steps, and transitions for a new workflow.

## Authentication

Requires Bearer token authentication. The authenticated user will be recorded as the workflow creator.

## Status Codes

<ResponseField name="201" type="Created">
  Workflow definition created successfully.
</ResponseField>

<ResponseField name="401" type="Unauthorized">
  Missing or invalid authentication token.
</ResponseField>

<ResponseField name="409" type="Conflict">
  A workflow with this key already exists.
</ResponseField>

<ResponseField name="422" type="Unprocessable Entity">
  Validation failed. Common validation errors:
  - Step codes must be unique
  - Exactly one start step is required
  - At least one terminal or end step is required
  - Transition source or target step does not exist
  - Custom action transitions require an actionCode
  - Unconditional self-loop detected
  - Step must have at least one outgoing transition (unless terminal)
  - Every step must be reachable from the start step
  - Cycle detected without maxVisitsPerInstance guard
  - Subworkflow step requires subworkflowMapping
</ResponseField>

## Validation Rules

### Workflow Structure

- **Unique step codes**: All step codes within a workflow must be unique
- **Single start step**: Exactly one step with `stepType: "start"` is required
- **Terminal steps**: At least one step must be terminal (`isTerminal: true` or `stepType: "end"`)
- **Reachability**: All steps must be reachable from the start step

### Transitions

- **Valid references**: Both `fromStepCode` and `toStepCode` must reference existing steps
- **Custom actions**: Transitions with `actionType: "custom"` must include an `actionCode`
- **Unconditional transitions**: Only one unconditional transition allowed per source step and action type
- **Self-loops**: Unconditional self-loops are not allowed
- **Outgoing transitions**: Non-terminal steps must have at least one outgoing transition

### Cycles

- **Loop guards**: Cycles (loops) are allowed but at least one step in the cycle must have `maxVisitsPerInstance` set to prevent infinite loops

### Subworkflows

- **Required mapping**: Steps with `stepType: "subworkflow"` must include a `subworkflowMapping` object
