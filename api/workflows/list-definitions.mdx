---
title: List Workflow Definitions
api: GET /api/v1/workflow-definitions
---

Retrieves a list of all workflow definitions with summary information including step and transition counts.

## Authentication

Requires Bearer token authentication.

## Response

<ResponseField name="items" type="array">
  Array of workflow definition summaries.
  
  <ResponseField name="items[].id" type="UUID">
    Unique identifier for the workflow definition.
  </ResponseField>
  
  <ResponseField name="items[].key" type="string">
    Unique key for the workflow definition.
  </ResponseField>
  
  <ResponseField name="items[].name" type="string">
    Human-readable name of the workflow.
  </ResponseField>
  
  <ResponseField name="items[].description" type="string">
    Description of the workflow.
  </ResponseField>
  
  <ResponseField name="items[].status" type="string">
    Current status: `draft`, `active`, or `archived`.
  </ResponseField>
  
  <ResponseField name="items[].latestVersionId" type="UUID | null">
    ID of the latest version, or null if no versions exist.
  </ResponseField>
  
  <ResponseField name="items[].latestVersionNo" type="integer | null">
    Version number of the latest version, or null if no versions exist.
  </ResponseField>
  
  <ResponseField name="items[].stepCount" type="integer">
    Total number of steps in the latest version.
  </ResponseField>
  
  <ResponseField name="items[].transitionCount" type="integer">
    Total number of transitions in the latest version.
  </ResponseField>
</ResponseField>

## Status Codes

<ResponseField name="200" type="OK">
  Workflow definitions retrieved successfully.
</ResponseField>

<ResponseField name="401" type="Unauthorized">
  Missing or invalid authentication token.
</ResponseField>

## Usage Notes

### Workflow Status

- **draft**: Workflow is being designed and not yet published
- **active**: Workflow is published and can be used to create instances
- **archived**: Workflow is no longer in active use but retained for historical purposes

### Sorting

Results are ordered by creation date in descending order (newest first).

### Version Information

- `latestVersionId` and `latestVersionNo` reflect the most recent version of the workflow definition
- Step and transition counts are based on the latest version
- If a workflow definition has no versions (edge case), these fields will be `null` and counts will be `0`

### Use Cases

1. **Workflow Directory**: Display all available workflows to users
2. **Workflow Selection**: Allow users to choose a workflow for creating instances
3. **Monitoring Dashboard**: Show overview of all workflow definitions with version info
4. **Admin Interface**: Manage and compare different workflow definitions
