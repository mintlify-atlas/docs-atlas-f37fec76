---
title: Get Workflow Definition
api: GET /api/v1/workflow-definitions/{definition_id}
---

Retrieves the complete details of a workflow definition including its latest version, all steps, transitions, and configuration.

## Authentication

Requires Bearer token authentication.

## Path Parameters

<ParamField path="definition_id" type="UUID" required>
  The unique identifier of the workflow definition to retrieve.
</ParamField>

## Response

<ResponseField name="id" type="UUID">
  Unique identifier for the workflow definition.
</ResponseField>

<ResponseField name="key" type="string">
  Unique key for the workflow definition.
</ResponseField>

<ResponseField name="name" type="string">
  Human-readable name of the workflow.
</ResponseField>

<ResponseField name="description" type="string">
  Detailed description of the workflow.
</ResponseField>

<ResponseField name="status" type="string">
  Current status of the workflow definition. One of: `draft`, `active`, `archived`.
</ResponseField>

<ResponseField name="latestVersionId" type="UUID">
  ID of the latest version of this workflow definition.
